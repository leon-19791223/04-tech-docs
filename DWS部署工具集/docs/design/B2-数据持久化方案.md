# B-2：数据持久化

## 1. 现状分析

| 问题 | 位置 | 影响 |
|:-----|:------|:------|
| Session 纯内存 | `core_engine.py` `_sessions = {}` | 重启后所有配置丢失 |
| 配置模板不持久 | 每次 `generate_templates()` 重新生成 | 无法回溯历史版本 |
| 预检结果不保存 | MOCK_PRECHECK 静态 | 无法做历史对比 |
| 无会话管理 UI | 无 | 用户只能用一个会话 |

## 2. 改造目标

| 维度 | 当前状态 | 目标状态 |
|:------|:---------|:---------|
| 会话持久化 | 内存 dict，重启丢失 | SQLite 后端，自动恢复 |
| 配置历史 | 无 | 每个会话保留配置快照 |
| 会话管理 | 无 | 列表/切换/删除 UI |
| 用户感知 | 无 | 侧边栏"会话管理"入口 |

**懒人友好设计**: 自动保存，用户无需手动操作。关闭页面自动持久化。

## 3. 改动文件清单

| 文件 | 操作 | 内容 |
|:-----|:------|:------|
| `engine/session_store.py` | **新增** | SQLite 持久化引擎 |
