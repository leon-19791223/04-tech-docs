# DWS 部署流程 — 官方文档 v.s. 工具集对比分析

> 官方文档: `嘉兴银行--DWS交付/DWS部署-V2.0.docx` + `GaussDB(DWS)测试集群标准实施交付文档.docx`  
> 工具集: `engine/core_engine.py` 8 阶段部署流 + `core/precheck_engine.py` 42项预检 + `core/verifier.py` 10项验证

---

## 一、官方部署完整流程

根据 `DWS部署-V2.0.docx`，实际的部署流程如下：

| 序号 | 阶段 | 步骤内容 | 对应工具集阶段 |
|:----:|:-----|:---------|:-------------:|
| 1 | 安装前检查 | 磁盘检查、网卡名一致性、网速规格、ISO存在性、服务器连通性、root密码一致性 | Phase 1/3 |
| 2 | 配置/etc/hosts+主机名 | 设置主机名 | Phase 1 |
| 3 | Python 3.8.5编译安装 | 下载源码->configure->make->altinstall | Phase 1 |
| 4 | 关闭auditd | systemctl stop/disable auditd | Phase 2 |
| 5 | 关闭防火墙 | firewalld + iptables 双关 | Phase 2 |
| 6 | OS内存水位线 | vm.watermark_scale_factor=100 | Phase 2 |
| 7 | Hi1822网卡处理 | hinicconfig增加中断个数 | ⚠️ **缺失** |
| 8 | RAID策略配置 | IBMC设置Read Ahead + Write Back With BBU | Phase 3 |
| 9 | NUMA参数 | kernel.numa_balancing=0 | Phase 2 |
| 10 | OOM策略 | panic_on_oom=0, oom_kill_allocating_task=0 | Phase 2 |
| 11 | 关闭透明大页 | grub追加transparent_hugepage=never | Phase 2 |
| 12 | 集群绑核(NUMA bind) | gs_guc reload enable_numa_bind=on | Phase 8 |
| 13 | 判断SSD/HDD | lsblk -d -o name,rota | Phase 3 |
| 14 | fio磁盘压测 | fio_arm - 8k/16k随机写 | Phase 3 |
| 15 | dd命令磁盘压测 | dd顺序写/读/读写 4k/8k/128k/1M | Phase 3 |
| 16 | 网络性能压测 | speed_test send/recv + sar带宽 + gsar重传率 | Phase 5 |
| 17 | 内存参数设置 | cstore_buffers (256G→1GB/8GB, 512G→2GB/16GB) | ⚠️ **缺失** |
| 18 | 并行参数设置 | query_dop=0, insert_dop=0 (POC开启) | ⚠️ **缺失** |
| 19 | 性能测试关闭参数 | autovacuum=off, audit_enabled=off, ssl=off | ⚠️ **缺失** |
| 20 | 关闭kerberos认证 | mppdb_inner_kerberos_enable=true→false | ⚠️ **缺失** |
| 21 | 磁盘挂载 | parted分区->mkfs.xfs->blkid->fstab->mount-a | Phase 4 |
| 22 | 挂载ISO+yum源 | mount ISO + 配置local.repo | Phase 1 |
| 23 | 安装依赖包 | gcc/libffi-devel/python-devel等 | Phase 1 |
| 24 | 时区设置 | timedatectl set-timezone Asia/Shanghai | Phase 2 |
| 25 | 关闭SELinux | sed -i 's/enforcing/disabled/' | Phase 2 |
| 26 | 关闭swap | swapoff + 注释fstab | Phase 2 |
| 27 | 查看字符集 | echo LANG=en_US.UTF-8 | Phase 2 |
| 28 | omm用户cron权限 | /etc/cron.allow + omm | ⚠️ **缺失** |
| 29 | 打开IPv6协议栈 | sysctl -a | grep ipv6|wc -l 检查 | ⚠️ **缺失** |
| 30 | vm.min_free_kbytes | free的5% | Phase 2(有但值不同) |
| 31 | IO调度策略 | elevator=deadline/mq-deadline | Phase 2 |
| 32 | 准备工具和软件 | scp GaussDB/FI安装包 | Phase 6 |
| 33 | 配置软件包 | unzip + tar + cp到对应目录 | Phase 6 |
| 34 | 配置preinstall.ini | g_hosts/g_user_name/g_parted等 | Phase 7 |
| 35 | 执行preinstall检查 | setuptool.sh preinstall | Phase 7 |
| 36 | 执行precheck检查 | setuptool.sh precheck | Phase 7 |
| 37 | 安装依赖RPM | haveged/sdparm/libatomic等 | Phase 7(有) |
| 38 | 安装双机Manager | install.sh -f xxx.ini (主+备) | Phase 8 |
| 39 | 更新sudo脚本 | optimization_patch.sh install/commit | ⚠️ **缺失** |
| 40 | 安装集群 | FusionInsight安装 | Phase 8(简略) |
| 41 | 加载License | TPOPS申请 | Phase 8(简略) |

## 二、差距分析

### 🔴 缺失项（官方有，工具集没有）

| # | 缺失内容 | 重要性 | 说明 |
|:-:|:---------|:------:|:------|
| M1 | **Hi1822网卡中断配置** | 高 | 官方文档明确指出"Hi1822网卡默认中断个数为16个，需通过hinicconfig工具增加"，当前工具集仅检测网卡存在性 |
| M2 | **内存参数设置(cstore_buffers)** | 高 | 官方根据内存大小（256G/512G）给出明确的cstore_buffers推荐值，直接影响列存查询性能 |
| M3 | **并行参数设置** | 中 | POC环境需设置query_dop=0，开启并行查询 |
| M4 | **性能测试关闭参数** | 中 | 测试前需关闭autovacuum/audit/ssl等，否则影响测试结果 |
| M5 | **关闭kerberos认证** | 中 | 非生产环境需关闭mppdb_inner_kerberos_enable |
| M6 | **omm用户cron权限** | 中 | 官方文档要求将omm加入/etc/cron.allow |
| M7 | **IPv6协议栈检查** | 低 | FusionInsight预检查的一部分 |
| M8 | **sudo脚本更新** | 高 | optimization_patch.sh install/commit 是安装过程中重要的前置步骤 |
| M9 | **安装后检查** | 高 | 5.1集群状态检查 + 5.2健康检查 + 5.3功能验证（标准实施文档有专项章节） |
| M10 | **License加载** | 中 | 生产环境需要，测试环境可跳过 |
| M11 | **dd盘性能测试详细记录** | 中 | 官方有dd.sh完整脚本，需记录到表（write/read/read&write各4种blocksize） |

### 🟡 不完整项（工具集有但需补充）

| # | 不完整内容 | 说明 |
|:-:|:-----------|:------|
| I1 | **Phase 2 vm.min_free_kbytes** | 官方公式：`free的5%`，工具集硬编码262144 |
| I2 | **Phase 3 fio测试** | 官方指定使用fio_arm（ARM编译版），目录为/srv/BigData/mppdb/data1，工具集写/dev/sdb |
| I3 | **Phase 3 dd测试** | 官方有完整dd.sh脚本（4种blocksize×3种模式），工具集仅一条命令 |
| I4 | **Phase 5 网络测试** | 官方使用speed_test工具+sar监控+gsar重传率，工具集缺少gsar重传率计算 |
| I5 | **Phase 7 preinstall.ini** | 官方ini有完整参数（oms_ip1/2, g_parted_conf, g_hostname_conf等），工具集生成器需对齐 |
| I6 | **Phase 8 OMS安装** | 官方有主/备管理节点分别安装的完整流程，工具集仅一条命令 |
| I7 | **Phase 8 集群绑核** | 官方在安装后单独执行gs_guc enable_numa_bind，工具集步骤正确但缺检查 |
| I8 | **验证项不足** | 官方验收测试指南含功能验证、健康检查、性能基线，verifier.py仅10项 |

## 四、结论

工具集当前 8 阶段覆盖了官方部署流程的 **约70%** 内容，主要缺失集中在：

1. **安装后验证**（最严重）— 缺少集群健康检查、功能验证、性能基线
2. **网卡专项配置** — Hi1822中断调整是华为ARM服务器的特有步骤
3. **内存和性能参数** — cstore_buffers/query_dop等对POC性能影响大
4. **sudo脚本更新** — 官方流程中部署前的必备步骤

建议优先补充 P0 项（安装后检查 + Hi1822网卡），其余依次推进。
