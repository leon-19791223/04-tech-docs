## 一、华为官方配置规划工具分析

### 1.1 工具定位与能力

`GaussDB(DWS) 9.1.0-ESL 配置规划工具`（V1.1.10）是华为官方提供的 Excel 宏工具，**不是仅配置生成，而是完整的部署规划平台**：

| 能力 | 说明 |
|:-----|:------|
| **容量规划** | 根据数据量/保留天数/增量，自动计算节点规格和数量 |
| **套餐选择** | 6种标准部署套餐（MN&CN合设/DN独立等）|
| **服务选择** | MPPDB/Monitor/DMSCollector/KrbServer/LdapServer |
| **IP规划** | 管理IP/业务IP/机架名称的完整规划 |
| **浮动IP** | OMServer/OMWebService 的浮动IP+接口+掩码+网关 |
| **磁盘规划** | OS盘 6分区 + 元数据盘 + 数据盘（精确到每个节点） |
| **节点信息** | CPU/内存/磁盘数/数据盘容量（按节点填写） |
| **集群参数** | MPPDB的dn/cms/gtm配置、透明加密、分区加密 |
| **实例参数** | DataNode/NameNode/NodeManager/ResourceManager 配置 |
| **配置导出** | 生成 preinstall.ini + host.ini + install_oms.ini + 集群安装模板 |

### 1.2 与工具集的对比

| 维度 | 华为配置规划工具 | 工具集配置生成器 | 差距 |
|:-----|:----------------|:----------------|:----:|
| 基础配置 | 8个必填字段 | 2个字段(cluster_name+IP) | 🔴 严重 |
| 服务选择 | 5种服务选择 | ❌ 无 | — |
| IP规划 | 管理IP/业务IP/机架 | 仅管理IP | 🔴 严重 |
| 浮动IP | 完整浮动IP配置 | ❌ 无 | — |
| 磁盘规划 | OS盘6分区+元数据+数据 | ❌ 无表格化展示 | 🟡 高 |
| 节点信息 | CPU/内存/磁盘/容量 | 部分有 | 🟡 中 |
| 集群参数 | 完整MPPDB参数 | ❌ 无 | — |
| 实例参数 | DN/NameNode参数 | ❌ 无 | — |
| 配置导出 | 官方g_格式完整 | 自创INI格式不兼容 | 🔴 严重 |
| **容量规划** | **根据数据量自动算节点** | ❌ **无** | 🔴 **缺失** |

### 1.3 关键结论

华为配置规划工具生成的 **output 文件列表**：

```
输出文件                              用途
──────────────────────────────────────────────────
preinstall.ini                        → setuptool.sh preinstall
preinstall/preinstall.ini             → SetupTool 预安装
preinstall/host0/1/2...ini           → 各节点分区配置
install_oms/*.ini                     → OMS Manager 安装配置
Cluster_Install_Template.xml         → Web GUI 集群安装模板
```

**工具集的 core/config_generator.py 输出格式与华为工具完全不兼容**，需要重写。

## 二、华为验收测试指南分析

### 2.1 测试覆盖体系

`GaussDB(DWS) 9.1.0-ESL 验收测试指南` 覆盖 **11 大测试类别**，按重要性排序：

| 类别 | 用例数 | 说明 | 工具集 verifier.py |
|:-----|:------:|:-----|:------------------:|
| **数据库基本功能** | 4 | 操作/查询/事务/存储过程 | 1项 (SELECT version) |
| **健康检查** | 5 | 集群/服务/主机/定时/报告 | 2项 (gs_om status+gs_checkperf) |
| **监控** | 4 | 主机/服务/看板/历史 | ❌ |
| **告警** | 4 | 阈值/查询/清除/导出 | ❌ |
| **备份恢复** | 2 | 表级别/集群级别 | 1项 (gs_dump备份) |
| **向导** | 1 | 启停操作 | ❌ |
| **配置** | 1 | 参数修改 | ❌ |
| **日志管理** | 1 | 筛选下载 | ❌ |
| **客户端** | 2 | 下载/安装 | ❌ |
| **用户管理** | 1 | 添加删除 | ❌ |
| **Coordinator实例** | 2 | 部署/功能 | ❌ |
| **应用访问IP** | 1 | 部署 | ❌ |
| | **~30+** | | **4项 (13%)** |

### 2.2 测试用例结构

每个测试用例格式：

```
┌─ 验收目的     → 一句话说明测试目标
├─ 预置条件     → 系统状态/配置前提
├─ 测试过程     → 1. 登录 → 2. 操作 → 3. 验证
├─ 预期结果     → 成功标准
├─ 测试结果     → 通过 / 失败
├─ 备注         → 针对不同环境(DEV/SIT/UAT/PREPROD)的差异
└─ 测试人/日期  → 签名
```

### 2.3 对比结论

**工具集的 verifier.py 当前仅覆盖了官方验收测试的约 13%**。如果需要工具集真正支撑交付验收，verifier 需要从 10 项扩展至 30+ 项。

## 四、最终优先级建议

基于 5 份文档的综合分析，将按以下优先级逐步优化：

### P0：核心功能不可用（4天）

| # | 任务 | 涉及官方文档 | 工时 | 说明 |
|:-:|:-----|:-----------|:----:|:-----|
| P0a | **Config Generator 重写** — 对齐华为配置规划工具的 preinstall.ini 输出格式 | 配置规划工具、部署V2.0 | 0.5天 | **当前格式完全不兼容，生成的配置无法使用** |
| P0b | **Verifier 扩展至30+项** — 覆盖验收测试指南的11大类 | 验收测试指南、标准实施文档 | 1.5天 | 当前覆盖率仅13%，需扩展至数据库功能/健康检查/监控/告警/备份等 |
| P0c | **补充Manager安装+集群创建** — 主备双节点OMS+Web GUI集群创建 | 标准实施文档、部署V2.0 | 1天 | 当前仅1条命令，缺少备节点安装和Web操作 |
| P0d | **补充缺失部署阶段** — Hi1822/cstore_buffers/sudo脚本/License | 部署V2.0、标准实施文档 | 1天 | 4个部署关键步骤缺失 |

### P1：交付能力提升（2天）

| # | 任务 | 涉及官方文档 | 工时 |
|:-:|:-----|:-----------|:----:|
| P1a | **配置规划增强** — 增加IP规划/浮动IP/磁盘规划表输出 | 配置规划工具 | 0.5天 |
| P1b | **容量规划** — 根据数据量/天数自动计算节点规模 | 配置规划工具 | 0.5天 |
| P1c | **验收报告模板** — 交付物增加验收测试报告（11类30+项） | 验收测试指南 | 0.5天 |
| P1d | **交付文档导出** — 一键导出规划+部署+验收全套文档 | 标准实施文档 | 0.5天 |

### P2：功能完善（1天）

| # | 任务 | 工时 |
|:-:|:-----|:----:|
| P2a | Bond配置管理（管理平面bond1+业务平面bond4） | 0.3天 |

| P2c | 环境预设扩展（稠州/绍兴/中银保信等） | 0.4天 |

### 整体工时汇总

| 批次 | 工时 | 交付物 |
|:-----|:----:|:-------|
| **P0** 🔴 | **4天** | 工具集核心功能可用 |

## 六、结论

| 评估维度 | 得分 | 一句话 |
|:---------|:----:|:-------|
| **配置规划** | 2/10 | 华为有完整规划工具，工具集配置生成器格式不兼容 |
| **部署流程** | 6/10 | 8阶段覆盖约65%，缺Manager安装/集群创建关键环节 |
| **预检检查** | 7/10 | 42项较完整，领先于华为配置工具的预检查 |
| **验收测试** | 2/10 | verifier.py覆盖13%，验收测试指南覆盖30+项 |
| **交付文档** | 4/10 | 有基础交付物，缺验收报告/规划表/运维指南 |
| **架构/展示** | 8/10 | 架构图/仪表盘/设备清单等UI优秀 |

**整体评分：4.8/10**

最核心的 3 个改善方向：
1. **preinstall.ini 格式对齐** — 否则生成的配置不可直接使用
2. **验收测试扩至 30+ 项** — 否则无法支撑正式交付验收
3. **补充 Manager 安装 + 集群创建** — 否则部署流程不完整
