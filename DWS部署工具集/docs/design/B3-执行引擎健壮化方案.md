# B-3：执行引擎健壮化

> **工时**: 1.5 天  
> **核心原则**: 不修改 `engine/core_engine.py` 一行

## 1. 现状分析

| 问题 | 位置 | 影响 |
|:-----|:------|:------|
| SSH 串行执行 | `ssh_executor.py:198` "暂不实现真并行" | 42项×3节点 = 126次串行，耗时数分钟 |
| 无连接复用 | 每次命令 `_connect()` + `_close()` | 握手开销大，可能被 SSHD 限流 |
| 超时硬编码 | `_exec_cmd()` timeout=30 | 部分命令（如 fio）可能需要更长时间 |
| 无重试机制 | 失败即返回 | 临时网络抖动导致误判 |
| 无部署回滚 | `core_engine.py` 步骤破坏后无法恢复 | 操作失误需手动修复 |

## 2. 改造目标

### 2.1 并行执行引擎

| 当前 | 目标 |
|:-----|:------|
| 串行执行 | ThreadPoolExecutor 真并行（max_workers=8） |
| 每命令新连接 | 连接池复用（max=10） |
| 超时 30s 硬编码 | 命令级可配置超时 |
| 无重试 | max_retries=2, 指数退避 |

### 2.2 部署回滚

为 `core_engine.py` 的 8 个部署阶段中破坏性操作提供回滚能力：

| 阶段 | 需回滚的步骤 | 备份内容 |
|:-----|:------------|:---------|
| 1. 环境准备 | chrony 配置 | `/etc/chrony.conf` |
| 2. OS 调优 | 所有 9 步 | `/etc/selinux/config`, `/etc/sysctl.conf`, `/etc/fstab`, grub 配置 |

## 3. 改动文件清单

| 文件 | 操作 | 内容 |
|:-----|:------|:------|
| `engine/connection_pool.py` | **新增** | SSH 连接池 |
| `engine/rollback_manager.py` | **新增** | 部署回滚管理器 |
| `engine/ssh_executor.py` | **修改** | 并行执行 + 重试 |
| `app.py` | **修改** | 集成回滚 + 并行配置 |

## 4. 懒人友好设计

- 并行执行自动并发，用户无需关心
- 部署出错自动提示"回滚上一步"，一键恢复
