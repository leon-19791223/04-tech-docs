# B-9：架构图重写方案

## 1. 问题回顾

| 问题 | 等级 | 说明 |
|:-----|:----:|:------|
| CSS变量全部错位 | P0 | `--bd`/`--ac`/`--tx`/`--t2`/`--s1` 在base.html中不存在 |
| SVG深色色板+浅色背景冲突 | P0 | `#e6edf3`近白文字在浅色背景看不见 |
| 合设场景可视化失败 | P0 | co_host浮动标签无法直观表达"一台机器跑多个角色" |
| 布局方向错误 | P1 | 按角色分层 vs 用户心智模型按物理节点 |
| 交互风格不统一 | P1 | 场景按钮/CNDN输入框使用自定义内联样式 |
| 无响应式 | P2 | SVG固定1480×540 |
| 信息层级扁平 | P2 | 统计/场景/硬件/图/图例权重相同 |

## 2. 改造方向

**从「SVG按角色分层」→「HTML卡片按物理节点分组」**

| 场景 | 节点数 | 布局方式 |
|:-----|:------:|:---------|
| DEV/SIT/UAT(合设) | 3 | 横向卡片排列，每卡显示所有角色 |
| PREPROD(分离) | 11 | 按角色分区网格：OM区→CN区→DN区→GTM区 |
| EXPAND(分离) | 15 | 同上 |

## 3. 改动文件

| 文件 | 操作 | 内容 |
|:-----|:------|:------|
| `app.py` | **修改** | api_architecture 增加 physical_nodes 分组 |
| `templates/dws_architecture.html` | **重写** | HTML卡片布局替代SVG |

## 4. API 扩展

```json
{
  "physical_nodes": [
    {
      "hostname": "dwsuat0001",
      "mgmt_ip": "10.134.21.190",
      "roles": ["OM","CN","DN","GTM"],
      "is_control": true,
      "virtual_roles": [{"id":"OM1","role":"OM"},{"id":"CN1","role":"CN"},...]
    }
  ]
}
```
