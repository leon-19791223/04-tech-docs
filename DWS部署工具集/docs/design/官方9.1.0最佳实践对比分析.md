## 二、与官方最佳实践的详细差距

### 2.1 【严重】preinstall.ini 格式完全不兼容

**官方格式**（FusionInsight SetupTool 实际需求）：

```ini
oms_ip1=10.134.21.190
oms_ip2=10.134.21.191
g_hosts="10.134.21.190,10.134.21.191,10.134.21.192"
g_user_name="root"
g_port=22
g_parted=3
g_parted_conf="10.134.21.190#host0.ini;10.134.21.191#host1.ini;..."
g_add_pkg=1
g_pkgs_dir="kylin-V10-SP2:/media"
g_log_file="/tmp/fi-preinstall.log"
g_debug=0
g_hostname_conf="10.134.21.190#10.134.21.190#uatdws0001;..."
g_swap_off=1
g_platform="aarch64"
g_core_dump=1
g_core_dump_dir="/var/log/core"
```

**工具集生成格式**（config_generator.py）：

```ini
[CLUSTER]
cluster_name = dws_cluster
master_node1_ip = 10.0.0.1
data_node_ips = 10.0.0.2,10.0.0.3
```

**问题**：格式和字段名完全不同。官方使用 `g_` 前缀的扁平键值对（FusionInsight SetupTool 规范），工具集使用 INI 节格式。生成的 preinstall.ini **不能直接用于 FusionInsight 部署**。

### 2.2 【严重】验收测试覆盖严重不足

官方验收测试指南覆盖以下完整测试类别：

| 测试类别 | 官方用例数 | 工具集 verifier.py |
|:---------|:---------:|:------------------:|
| 集群健康检查 | 5+ | 1（基础gs_om status）|
| 服务/角色监控 | 4 | 0 |
| 告警功能 | 4 | 0 |
| 配置管理 | 2 | 0 |
| 数据库基本功能 | 4 | 1（SELECT 1）|
| 备份恢复 | 4 | 1（gs_dump）|
| 客户端安装使用 | 2 | 0 |
| 用户管理 | 1 | 0 |
| Coordinator实例 | 2 | 0 |
| 应用访问IP | 1 | 0 |
| 性能基线 | 3 | 2（fio/网络）|
| **总计** | **~40+** | **10** |

### 2.3 【严重】部署阶段与官方流程不匹配

| 官方阶段 | 工具集对应 | 差异 |
|:---------|:----------|:-----|
| 安装前检查（磁盘/网卡/ISO/连通性） | Phase 1/3 | 大部分覆盖，缺Hi1822中断 |
| Python 3.8.5环境准备 | Phase 1 | 有 |
| OS配置（audit/防火墙/THP/SELinux/swap等） | Phase 2 | 基本覆盖 |
| 内存水位线+NUMA+OOM | Phase 2 | 有 |
| **Hi1822网卡中断配置** | ❌ **缺失** | 关键，官方强调处理 |
| **RAID策略配置** | Phase 3 | 有检查无配置 |
| **fio/dd磁盘压测** | Phase 3 | 有但命令与官方不完全一致 |
| **网络性能压测(speed_test+gsar)** | Phase 5 | 缺gsar重传率 |
| **挂载ISO+yum源** | Phase 1 | 有 |
| **磁盘分区+格式化+fstab** | Phase 4 | 有 |
| **cstore_buffers内存参数** | ❌ **缺失** | 直接影响性能 |
| **并行参数设置(query_dop)** | ❌ **缺失** | POC性能关键 |
| **性能测试关参数** | ❌ **缺失** | autovacuum/audit/ssl等 |
| **关闭kerberos** | ❌ **缺失** | 非生产环境必需 |
| **omm用户cron权限** | ❌ **缺失** | 预检查通过条件 |
| **配置preinstall.ini** | ⚠️ 格式不兼容 | 见2.1 |
| **执行preinstall/precheck** | Phase 7 | 有但命令需对齐 |
| **安装依赖RPM** | Phase 7 | 有 |
| **安装双机Manager** | Phase 8 | 缺少备节点步骤 |
| **更新sudo脚本** | ❌ **缺失** | 官方必须步骤 |
| **集群绑核(NUMA bind)** | Phase 8 | 有 |
| **License加载** | ❌ **缺失** | 生产环境必需 |
| **安装后验证** | ❌ **严重缺失** | 见2.2 |

### 2.4 【中】预检项覆盖面不足

当前 `precheck_engine.py` 42项（硬件12/OS15/网络8/存储5/软件2）：

| 类别 | 现有 | 建议增加 |
|:-----|:----:|:---------|
| 硬件 | 12 | Hi1822中断个数检查、BIOS 7项自动化检查 |
| OS | 15 | IPv6协议栈检查、cron.allow权限、内核版本核对 |
| 网络 | 8 | 无显著缺失 |
| 存储 | 5 | 无显著缺失 |
| 软件 | 2 | FusionInsight安装包完整性检查 |
| **新增类别** | - | **部署后验证（20+项）** |

### 2.5 【中】配置生成器需对齐官方 preinstall.ini

官方 preinstall.ini 字段：

| 字段 | 含义 | 工具集状态 |
|:-----|:-----|:----------:|
| `oms_ip1/oms_ip2` | 主备OM IP | ❌ 缺失 |
| `g_hosts` | 所有节点IP列表 | ⚠️ 有但格式不同 |
| `g_user_name` | SSH用户名 | ❌ 缺失 |
| `g_port` | SSH端口 | ❌ 缺失 |
| `g_parted` | 分区模式(1自动/2手动/3跳过) | ❌ 缺失 |
| `g_parted_conf` | 各节点分区配置 | ❌ 缺失 |
| `g_add_pkg` | 是否添加依赖包 | ❌ 缺失 |
| `g_pkgs_dir` | yum源路径 | ❌ 缺失 |
| `g_hostname_conf` | IP#IP#主机名映射 | ❌ 缺失 |
| `g_swap_off` | 是否关闭swap | ❌ 缺失 |
| `g_platform` | 平台架构 | ❌ 缺失 |
| `g_core_dump` | 是否开启coredump | ❌ 缺失 |

## 五、结论

| 维度 | 得分 | 说明 |
|:-----|:----:|:------|
| 部署流程覆盖度 | ⚠️ 6/10 | 8阶段覆盖约70%，缺Hi1822/cstore_buffers/sudo/License等 |
| preinstall.ini 兼容性 | ❌ 0/10 | 格式完全不兼容，不可直接使用 |
| 验收测试覆盖度 | ❌ 2/10 | verifier.py仅10项，官方40+项 |
| 预检覆盖度 | ✅ 7/10 | 42项基本合理，可扩展至48项 |
| 配置规划 | ⚠️ 6/10 | 参数覆盖广但输出格式不对 |
| **整体** | **⚠️ 4/10** | 需P0修复后方可投入正式使用 |

**最优先的3件事**：
1. **重写 `config_generator.py`** — preinstall.ini 对齐官方格式（0.5天）
2. **扩展 `verifier.py`** — 从10项到40+项验收测试（1天）
3. **补充缺失部署阶段** — 补齐 Hi1822/cstore_buffers/sudo脚本等（1天）
