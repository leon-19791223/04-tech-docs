## 工作流程

每个开发步骤遵循以下标准流程：

```
┌─────────────────────────────────────────────────────────┐
│                 单步工作流（每步一次循环）                  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Step 1: git tag pre-<step>      ← 保存当前版本快照       │
│  Step 2: 编写设计文档              ← 含改什么/怎么改/影响范围 │
│  Step 3: 方案评审                  ← 确认设计无误后实施     │
│  Step 4: 实施编码                  ← 按设计文档执行         │
│  Step 5: 测试验证                  ← 单元测试/端到端/回归  │
│  Step 6: git tag v-<step>         ← 验证通过后打标       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Git 标签命名

| 标签类型 | 格式 | 示例 | 说明 |
|:---------|:-----|:-----|:------|
| **Pre-tag** | `dws-bX-prep` | `dws-b1-prep` | 实施前快照，绝对回滚点 |
| **验证 tag** | `dws-bX-done` | `dws-b1-done` | 实施+测试通过后的里程碑 |

### 测试清单（每步通用）

以下测试每步实施完成后必须执行：

- [ ] **代码检查**：`git diff --stat` 确认改动的文件符合预期
- [ ] **语法检查**：`python -m py_compile <改动的文件>` 无错误
- [ ] **启动测试**：`python app.py` 启动不报错，`curl /api/health` 返回 200
- [ ] **回归测试**：原有功能不受影响（主页/预检/配置/架构图/审计/环境切换）
- [ ] **回滚验证**：`git restore --source=<pre-tag> -- DWS部署工具集/` 可恢复

## 步骤 B-1：SSH 安全加固

> **设计文档**: `docs/design/B1-SSH安全加固方案.md`  
> **工时**: 1.0 天  
> **目标**: 消除 SSH 连接中的安全风险（MITM、明文凭据、无审计）

### 1.1 设计内容

| 改造点 | 当前状态 | 目标状态 |
|:--------|:---------|:---------|
| 主机密钥验证 | `AutoAddPolicy()` 自动接受 | 三等级策略（strict/warn/insecure） |
