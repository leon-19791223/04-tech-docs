📋 TPOPS安装后的完整流程

Z-DBMate 完成 TPOPS 部署后，需按以下步骤在 TPOPS Web 界面完成 GaussDB 实例安装。共 5 个阶段：

1️⃣ License申请 TPOPS Web界面
  1. 登录 TPOPS：https://<Web-IP>:8002/gaussdb/
  2. 默认账号：admin / Gauss_246（首次登录需修改密码）
  3. 进入 平台管理 → License管理 → 许可容量 → 申请试用
  4. 试用有效期 60 天，到期后需申请正式 License
⚠ 未配置License无法安装实例，必须先申请试用或导入正式License
2️⃣ 存储设备配置（可选） TPOPS Web界面
  1. 进入 平台管理 → 存储设备 → 添加存储设备
  2. 选择类型：NAS
  3. 填写NAS服务器地址和共享目录路径（必须以 / 开头）
  4. 用于数据库备份存储，如不配置不影响实例安装
✅ 未添加存储设备不影响实例安装与使用，但建议生产环境配置
3️⃣ 添加主机 TPOPS Web界面
⚠ 添加主机前确保 GaussDB 节点已完成磁盘分区（手动操作）

3.1 GaussDB节点磁盘分区（手动）

LV名称大小挂载点说明
lv_ob_cluster200G/data/cluster集群主目录
lv_ob_etcd200G/data/cluster/usr/local/etcdETCD数据
lv_ob_data500G+/data/cluster/var/lib/engine/data1/data数据库数据
lv_ob_log500G/data/cluster/var/lib/log数据库日志
创建命令示例: 
pvcreate /dev/nvme0n1 /dev/nvme1n1
vgcreate vg_data /dev/nvme0n1
lvcreate -L 200G --name lv_ob_cluster vg_data
lvcreate -L 200G --name lv_ob_etcd vg_data
lvcreate -L 500G --name lv_ob_data vg_data
lvcreate -L 200G --name lv_ob_log vg_data
mkfs.ext4 /dev/mapper/vg_data-lv_ob_cluster
mkfs.ext4 /dev/mapper/vg_data-lv_ob_etcd
mkfs.ext4 /dev/mapper/vg_data-lv_ob_data
mkfs.ext4 /dev/mapper/vg_data-lv_ob_log
mount /dev/mapper/vg_data-lv_ob_cluster /data/cluster
mkdir -p /data/cluster/usr/local/etcd
mkdir -p /data/cluster/var/lib/engine/data1/data
mkdir -p /data/cluster/var/lib/log
mount /dev/mapper/vg_data-lv_ob_etcd /data/cluster/usr/local/etcd
mount /dev/mapper/vg_data-lv_ob_data /data/cluster/var/lib/engine/data1/data
mount /dev/mapper/vg_data-lv_ob_log /data/cluster/var/lib/log
echo "/dev/mapper/vg_data-lv_ob_cluster /data/cluster ext4 nodev,nosuid,noatime,nofail 0 1" >> /etc/fstab
# ... 其他挂载点同理

3.2 TPOPS添加主机

  1. 登录 TPOPS → 平台管理 → 数据中心管理 → 选择机房
  2. 点击 添加主机，填写GaussDB节点信息
  3. 确认信息后点击 确定，等待 5-10 分钟
  4. 前往 任务中心 查看执行情况，失败可重试
4️⃣ 安装实例 TPOPS Web界面
  1. 实例安装前确保所有主机已添加成功并初始化完成
  2. 进入 实例管理 → 安装实例
  3. 选择配置：
    • 实例类型：主备版（集中式）
    • 部署形态：3节点
    • 副本一致性协议：Paxos
    • 数据库端口：自定义（默认40080）
    • 管理员密码：设置 DBA(root)密码
    • 参数模板：可选（或安装后再修改参数）
  4. 选择已添加的主机节点
  5. 确认信息后点击 立即申请，提交等待任务流完成
✅ 安装成功后，可通过 gsql -d postgres -h {IP} -p {port} -U root 连接数据库验证
5️⃣ 部署后配置 推荐执行

5.1 备份策略配置

  1. 实例管理 → 点击实例 → 备份恢复 → 修改备份策略
  2. 根据业务需求配置备份周期和保留策略
  3. ⚠️ 备份策略开启后无法关闭，仅能修改参数

5.2 实例参数优化（建议）

参数名建议值说明
enable_global_plancacheon开启全局计划缓存
behavior_compat_options'bind_procedure_searchpath,truncate_numeric_tail_zero,...'兼容性选项
audit_enabledon开启审计日志
max_connections按需最大连接数

可在 TPOPS → 实例管理 → 参数管理 中批量修改，建议每次修改小于30个参数。

5.3 安装gsql客户端（可选）

# 从 DBS-GaussDB-driver 包中解压
mkdir -p /usr/local/gsql
cd /usr/local/gsql
tar -zxvf GaussDB-Kernel_505.1.0_Kylin_64bit_Gsql.tar.gz
# 配置环境变量
echo 'export PATH=/usr/local/gsql/bin:$PATH' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=/usr/local/gsql/lib:$LD_LIBRARY_PATH' >> ~/.bashrc
source ~/.bashrc
# 连接测试
gsql -d postgres -h 192.168.0.11 -p 40080 -U root

5.4 监控与DBMind（可选）

  1. 如需安装DBMind分析服务器，在TPOPS → DBMind管理 → 创建DBMind实例
  2. DBMind支持ARM和X86服务器（24.1.31版本以上）
  3. 创建后可在 DBMind管理 中纳管数据库实例
